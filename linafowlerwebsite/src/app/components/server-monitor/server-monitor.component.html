Server Monitor

Project Brief:

HealthEquity has a series of servers that were used by the SCRUM teams for their automation testing.

The problem we were trying to solve was the agents would go offline unexpectedly and wouldn't come back online automatically. Also, there wasn't a way for our users to restart the agent after hours.

Our goal was to create a web application that watched the agent status. When the agent went offline the application would bring it back up. Also, the application needed to allow our users to restart the agent when needed.

MY PROCESS

Research

Interview users to get a better understanding of the problem we are trying to solve.

Define

Create personas, role objectives, decisions, and understand user pain points.

Ideate

Create sketches, low fidelity wireframes, user flows, and gather user feedback.

Design

Create detailed designs, style guide, stakeholder reviews, and gather user feedback.

Test

Live prototype is created and begin user testing. Gather user feedback.

Research & Define

Before we began the design, I first spoke with the users to determine what pain points they were experiencing.

After research was gathered, I created personas to help keep the user as the center of the design.

Ideation

In the ideation phase I created discussed with my team creative solutions.

After that I created wireframes and determined the basic outline of what the application would start to look like.

Design

Once we had the wireframes built we had our foundation down for the product.

At which point I begun starting on the design. Creating high-fidelity mocks before building it out into code.

>

Testing

Before deploying to production we tested our product with our users through group testing and one on one interviews.

However, this is where I would have done more testing to mitigate any confusion for the user before getting to this stage.

Reflection

What I have learned throughout the design process for this product is to do more adequate testing early on in the process. Making sure to gather feedback at each stage is crucial to creating a good product and ensuring you're focusing it around the user and the problem you're trying to solve.

Also, never assume the user knows what the application is for and make sure you are on the same understanding. Basically getting a prototype in front of the user early on is the best approach.
